# 实验报告

## 一、实验题目

基于随机滑动 FrozenLake 的 Q-learning 求解实验

## 二、实验目的

1. 理解 Gymnasium 中 FrozenLake 环境的状态、动作、奖励与终止机制。
2. 掌握表格型 Q-learning 的基本实现方法。
3. 观察在随机滑动条件下，探索机制对学习效果的影响。
4. 比较在 8×8 预设地图与自定义 12×12 地图上，Q-learning 的训练难度与最终表现差异。

## 三、实验环境

- **操作系统**：Windows 11
- **Python 版本**：Python 3.10（conda 环境 `rl_course`）
- **核心依赖**：Gymnasium 1.2.3、NumPy 2.2.6、Matplotlib 3.10.8、Pillow

## 四、环境描述

### 4.1 FrozenLake-v1 环境

FrozenLake-v1 是 Gymnasium Toy Text 中的经典离散网格世界环境。智能体在一个冰冻湖面上移动，目标是从起点 **S** 到达终点 **G**，同时避免掉入冰洞 **H**。

- **状态空间**：离散，状态数 = 行数 × 列数（8×8 = 64 个状态，12×12 = 144 个状态）
- **动作空间**：离散 4 个动作 — 0（左）、1（下）、2（右）、3（上）
- **奖励机制**：到达终点 G 获得奖励 1.0，其余情况奖励为 0.0
- **终止条件**：到达 G（成功）或掉入 H（失败），或超过最大步数（截断）

### 4.2 随机滑动机制

本实验中所有环境均设置 `is_slippery=True`。在该模式下，智能体执行动作时不一定沿期望方向移动：

- 以 **1/3** 的概率朝期望方向移动
- 以 **1/3** 的概率朝顺时针垂直方向移动
- 以 **1/3** 的概率朝逆时针垂直方向移动

这使得环境具有很强的随机性，即便学到最优策略，成功率也无法达到 100%（尤其在陷阱密集的地图上）。

## 五、算法原理

### 5.1 Q-learning 算法

Q-learning 是一种无模型（model-free）的离策略（off-policy）时序差分（TD）强化学习算法。其核心思想是维护一个 Q 表 $Q(s,a)$，记录在状态 $s$ 下执行动作 $a$ 的期望累积回报。

**核心更新公式**：

$$Q(s,a) \leftarrow Q(s,a) + \alpha \left[ R + \gamma \max_{a'} Q(s',a') - Q(s,a) \right]$$

其中：
- $\alpha$ 为学习率，控制新信息的接受程度
- $\gamma$ 为折扣因子，衡量未来奖励的重要性
- $R$ 为即时奖励
- $\max_{a'} Q(s',a')$ 为下一状态的最优 Q 值估计

### 5.2 epsilon-greedy 探索策略

为平衡探索（exploration）与利用（exploitation），采用 $\varepsilon$-greedy 策略：

- 以概率 $\varepsilon$ 随机选择动作（探索）
- 以概率 $1 - \varepsilon$ 选择当前 Q 值最大的动作（利用）

$\varepsilon$ 随训练进行按指数衰减：$\varepsilon \leftarrow \max(\varepsilon_{\min}, \varepsilon \times \varepsilon_{\text{decay}})$

### 5.3 乐观初始化

将 Q 表初始值设为 1.0（而非 0.0）。由于 FrozenLake 的实际奖励最大为 1.0，乐观的初始值会促使智能体在早期更积极地探索未访问过的状态——因为未被更新过的状态-动作对的 Q 值偏高，智能体倾向于去尝试这些"看起来很好"的未知区域，从而加速在大地图上发现终点。

## 六、核心代码实现

### 6.1 Q-learning 智能体

```python
class QLearningAgent:
    def __init__(self, n_states, n_actions, alpha=0.1, gamma=0.99,
                 epsilon=1.0, epsilon_min=0.01, epsilon_decay=0.9995,
                 q_init=0.0):
        self.q_table = np.full((n_states, n_actions), q_init, dtype=float)
        # ... 其余参数赋值

    def choose_action(self, state):
        """epsilon-greedy 动作选择"""
        if np.random.random() < self.epsilon:
            return np.random.randint(self.n_actions)
        return int(np.argmax(self.q_table[state]))

    def update(self, state, action, reward, next_state, done):
        """Q-learning 更新"""
        td_target = reward + self.gamma * np.max(self.q_table[next_state]) * (1 - done)
        td_error = td_target - self.q_table[state, action]
        self.q_table[state, action] += self.alpha * td_error
```

### 6.2 训练过程

```python
def train(env, agent, n_episodes, log_interval=2000):
    for ep in range(n_episodes):
        state, _ = env.reset()
        done = False
        while not done:
            action = agent.choose_action(state)
            next_state, reward, terminated, truncated, _ = env.step(action)
            done = terminated or truncated
            agent.update(state, action, reward, next_state, float(terminated))
            state = next_state
        agent.decay_epsilon()
```

### 6.3 测试过程

```python
def test(env, agent, n_episodes=1000):
    for _ in range(n_episodes):
        state, _ = env.reset()
        done = False
        while not done:
            action = int(np.argmax(agent.q_table[state]))  # 纯贪婪
            next_state, reward, terminated, truncated, _ = env.step(action)
            done = terminated or truncated
            state = next_state
```

## 七、实验参数

| 参数 | 8×8 地图 | 12×12 地图 | 说明 |
|------|---------|-----------|------|
| 学习率 $\alpha$ | 0.15 | 0.15 | 控制每次更新幅度 |
| 折扣因子 $\gamma$ | 0.99 | 0.99 | 重视长远回报 |
| 初始探索率 $\varepsilon$ | 1.0 | 1.0 | 初始完全随机探索 |
| 最小探索率 $\varepsilon_{\min}$ | 0.02 | 0.02 | 保留少量探索 |
| 探索衰减 $\varepsilon_{\text{decay}}$ | 0.99995 | 0.99998 | 12×12 衰减更慢 |
| Q 表初始值 | 1.0 | 1.0 | 乐观初始化 |
| 训练回合数 | 50,000 | 100,000 | 大地图需更多训练 |
| 最大步数/回合 | 200（默认）| 500 | 大地图需更多步 |

**参数选择说明**：

- $\varepsilon_{\text{decay}}$ 选择了比常规建议值（0.995~0.9995）更慢的衰减速率。原因是在 `is_slippery=True` 的大地图上，智能体需要很长时间才能通过随机探索首次到达终点。如果 $\varepsilon$ 衰减太快，智能体在发现终点之前就停止了探索，导致 Q 表从未被正向奖励更新过，训练彻底失败（成功率 0%）。
- 12×12 地图的衰减速率比 8×8 更慢（0.99998 vs 0.99995），因为更大的状态空间需要更长的探索周期。
- `max_episode_steps` 为 12×12 地图设置为 500，因为从起点到终点的最短曼哈顿距离为 22 步，在随机滑动下实际路径远长于此，200 步的默认限制会导致大量回合被截断。

## 十、结果分析

### 10.1 两种地图的训练难度对比

| 对比维度 | 8×8 地图 | 12×12 地图 |
|----------|---------|-----------|
| 状态空间大小 | 64 | 144 |
| 冰洞数量 | 6 | 11 |
| 陷阱密度 | 6/64 = 9.4% | 11/144 = 7.6% |
| 首次发现终点所需探索量 | 较少 | 较多 |
| 训练开始见效回合 | ~5,000 | ~20,000 |
| 所需总训练回合 | 50,000 | 100,000 |

12×12 地图的状态空间是 8×8 的 2.25 倍，但所需训练回合数为 2 倍，说明训练量基本与状态空间规模成正比。训练开始见效的回合则差异更大（约 4 倍），这是因为随机探索到达终点的概率随距离呈指数下降。

### 10.2 测试性能分析

8×8 地图的测试成功率为 **56.90%**，而 12×12 地图达到了 **99.90%~100%**。这个看似反直觉的结果可以从以下角度解释：

1. **陷阱密度差异**：8×8 地图的陷阱密度（9.4%）高于 12×12 地图（7.6%），且 8×8 的陷阱分布更加集中，在终点附近形成了密集的"陷阱带"（第 5~7 行共有 6 个 H），即使策略正确，随机滑动也极易落入冰洞。
2. **路径宽度**：12×12 地图由于面积更大，安全路径更多更宽，即使发生滑动偏移，也更可能落在安全的冰面上而非冰洞中。
3. **最大步数**：12×12 的 `max_episode_steps=500` 给予了智能体更多的纠错机会，即使滑偏也有时间修正回正确路径。

### 10.3 随机滑动对策略的影响

在确定性环境（`is_slippery=False`）中，最优策略通常沿最短路径直线移动。但在随机滑动条件下，策略呈现了以下特征：

- **避险倾向**：在冰洞附近，策略倾向于选择"平行于冰洞"的方向而非"朝向冰洞"的方向，以降低滑入冰洞的概率。
- **路径冗余**：策略并非总是选择最短路径，而是选择容错率更高的路径。
- **方向多样性**：即使在远离冰洞的区域，策略方向也不完全一致，反映了在随机环境下多条路径的 Q 值差距不大。

### 10.4 乐观初始化的作用

实验中使用了 Q 表乐观初始化（$Q_{\text{init}} = 1.0$）。与零初始化对比：

- **零初始化 + 常规衰减**（$\varepsilon_{\text{decay}} = 0.9995$）：在两种地图上训练完全失败，成功率 0%。因为 Q 表全零且 FrozenLake 只在终点给予奖励，智能体在 $\varepsilon$ 降低之前从未到达过终点，导致贪婪策略完全基于未经更新的零值 Q 表，本质上是随机选择。
- **乐观初始化**：未被访问过的状态-动作对保持高 Q 值，贪婪选择会倾向于尝试这些"看起来很好"的未知动作，实现了一种隐式的探索激励，即使在 $\varepsilon$ 较低时也能继续探索。

## 十一、结论

1. 表格型 Q-learning 能够有效求解 `is_slippery=True` 的 FrozenLake 环境，在 8×8 地图上达到约 57% 的成功率，在 12×12 自定义地图上达到近 100% 的成功率。
2. 随机滑动显著增加了训练难度，不仅需要更多的训练回合，还要求更加精心的超参数调节（特别是探索衰减速率和 Q 表初始化策略）。
3. 地图大小并非决定最终性能的唯一因素——陷阱密度和分布对成功率的影响更为关键。8×8 地图虽小，但陷阱密集，随机滑动下容错空间极小；12×12 地图虽大，但陷阱稀疏，路径丰富。
4. 乐观初始化在稀疏奖励的大状态空间环境中至关重要，它为智能体提供了持续探索的内在动力，弥补了 $\varepsilon$-greedy 在这类环境中的不足。
5. 探索策略的设计（$\varepsilon$ 衰减速度、乐观初始化）对大地图的训练成败起决定性作用，不合理的探索策略会导致训练完全失败。
